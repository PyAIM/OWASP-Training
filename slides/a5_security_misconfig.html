A5: Security Misconfiguration

Exploiting improperly configured security settings

What is Security Misconfiguration?

Security Misconfiguration occurs when security settings are defined, implemented, or maintained improperly. This can include using default configurations, incomplete setups, open cloud storage, error messages that reveal too much information, and more.
Default Credentials
Using default or weak passwords on systems and applications
Verbose Errors
Error messages revealing sensitive implementation details

Finding Misconfigurations

  • Check for default credentials on admin interfaces
  • Look for directory listings that expose sensitive files
  • Search for backup files with extensions like .bak, .old, .backup
  • Test for verbose error messages that reveal too much information
