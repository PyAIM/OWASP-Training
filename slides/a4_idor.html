A4: Insecure Direct Object References

Manipulating object references to access unauthorized data

What is IDOR?

Insecure Direct Object Reference (IDOR) occurs when an application provides direct access to objects based on user-supplied input. This allows attackers to bypass authorization and access resources they shouldn't have permission to access.
URL Manipulation Example
https://example.com/account?id=123 → https://example.com/account?id=124

Exploitation Steps

1
Log in with a valid account
2
Identify object references in URLs
3
Modify the reference values
4
Access unauthorized resources
Common IDOR Targets
  • User profiles and account information
  • Shopping carts and order details
  • Documents and files
  • API endpoints with numeric IDs
CTF Challenge
Objective: Access another user's basket by manipulating the basket ID.
Hint: Add an item to your basket, then observe and modify the basket ID in the URL.
http://localhost:3000/rest/basket/1 → http://localhost:3000/rest/basket/2
