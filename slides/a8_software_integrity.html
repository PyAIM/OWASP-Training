A8: Software and Data Integrity Failures

Exploiting vulnerabilities in the software supply chain

What are Software Integrity Failures?

Software and data integrity failures occur when code and infrastructure don't protect against integrity violations. This includes using untrusted software, libraries, or modules from sources with weak integrity protection, and can lead to serious supply chain attacks.
CI/CD Pipeline Attacks
Compromising build and deployment systems
Dependency Confusion
Tricking package managers to use malicious packages

Supply Chain Attacks

Supply chain attacks target less-secure elements in the software supply network. By compromising a single supplier, attackers can distribute malicious code to all organizations using the supplier's products, as seen in the SolarWinds attack.
